# 表面码 (Surface Code)

> 来源批注：*"a distance-5 surface code"* — Bluvstein et al., 2026, p.40
> 参考文献：高健硕，"Surface Code VS Color Code"，2025（位于 `Rydberg atom/supplementary/`）
> 本笔记从稳定子形式理论出发，介绍表面码的定义、错误检测机制、逻辑算子和解码方法。与 [[Color-Code|颜色码]] 对比时，表面码的核心特征是**只支持近邻连接、只需两种稳定子（X 和 Z）**。

---

## 1. 什么是表面码？

表面码是目前**最实用的量子纠错码**之一，因为它：
- 只需要**近邻量子比特之间的二体相互作用**（易于实验实现）
- 具有约 $1\%$ 的高错误阈值（$\approx 10^{-2}$）
- 逻辑错误率随码距指数衰减

表面码将 qubit 排列在一个 **$d \times d$ 的二维网格**上（$d$ 为码距）。

> [!tip] 物理直觉
> 想象一个二维棋盘格，每个交叉点放一个量子比特。某些方格（plaquette）测量 Z，某些顶点（vertex）测量 X。如果某个比特出错了，相邻的稳定子测量会"报警"——就像围棋中一个子落错位置会让周围的裁判发现异常。

## 2. 物理结构

以 $d = 3$ 表面码为例（共 $d^2 + (d-1)^2 = 13$ 个物理 qubit）：

```
● ─ ○ ─ ●
│   │   │
○ ─ ● ─ ○
│   │   │
● ─ ○ ─ ●
```

- **数据 qubit（●）**：存储逻辑信息
- **辅助 qubit（○）**：用于测量稳定子（syndrome 提取）

> [!tip] 交错棋盘格布局
> 注意 ● 和 ○ 呈**棋盘格交错排列**——每个辅助 qubit 被数据 qubit 包围，可同时检测周围 4 个 qubit 的错误。这种布局是表面码只需近邻连接的几何基础。

### 2.1 Toric Code（环面码）基础

一般表面码定义在 **$L \times L$ 的正方形网格**上，量子比特位于网格的边上。如果把这个网格卷成环面（torus），就得到 **Toric Code**。后面 [[Color-Code|颜色码]] 也是基于同样的拓扑原理。

对于 Toric Code：$n = 2L^2$ 个 qubit，$m = 2L^2 - 2$ 个独立稳定子，因此逻辑比特数 $k = n - m = 2$，即环面编码了 **2 个逻辑比特**。

## 3. 稳定子结构

表面码有两类稳定子（参见 [[QEC]]）：

**X 型稳定子**（检测 Z 错误）——作用在顶点 $v$ 上：
$$
A_v = \prod_{e \in \text{vertex } v} X_e
$$

**Z 型稳定子**（检测 X 错误）——作用在方块 $p$ 上：
$$
B_p = \prod_{e \in \text{plaquette } p} Z_e
$$

两组稳定子的测量结果共同给出错误综合征（syndrome），解码器据此推断并修复错误。

> [!info] X 和 Z 稳定子是对偶的
> 如果把每个方块的中心点上一个点，再把这些新的点作为网格点，就会发现 X 和 Z 稳定子结构完全对偶——X 稳定子检测 Z 类型的错误链，Z 稳定子检测 X 类型的错误链。

### 3.1 稳定子形式理论

稳定子码可以用**投影算符**定义：

$$
\Pi_C = \frac{1}{2^m} (I + S_1)(I + S_2) \cdots (I + S_m)
$$

其中 $S_i \in \mathcal{S}$ 是编码空间上的稳定子生成元。关键性质：

- 对编码空间：$\Pi_C |\psi\rangle = |\psi\rangle$（$|\psi\rangle \in C$）
- 对非编码空间：$\Pi_C |\psi\rangle = 0$（$|\psi\rangle \notin C$）
- 编码空间维度：$\dim(C) = \text{Tr}[\Pi_C] = 2^{n-m}$

> [!warning] 为什么 Tr 只贡献单位算符
> $\Pi_C$ 遍历了稳定子群中的所有元素，而其中包含泡利矩阵成分的元素求迹后均为 0（泡利矩阵的迹为零）。因此最终只有单位算符贡献——这是利用了泡利矩阵的 traceless 性质。

## 4. 错误检测：错误链与端点

> [!tip] 物理直觉：错误像一条"路径"
> X 型错误会在一条数据 qubit 链上翻转比特。稳定子测量只会在**错误路径的端点**检测到异常（syndrome）。中间的错误互相抵消——这就像一条路上有两个人同时翻转，只在两头露出破绽。

以 X 型错误为例：症状检测只会在错误路径的**端点**检测到异常。两个异常点之间的距离就是**曼哈顿距离**（Manhattan distance），这直接决定了解码器需要搜索的最小错误链。

## 5. 逻辑算子：非平凡环路

> [!tip] 物理直觉：拓扑保护
> 稳定子测量只能检测"可以缩成一个点的环路"（平凡环路）。如果一条错误路径形成了**无法缩成点的环路**（非平凡环路），稳定子就探测不到——这就是逻辑算子。

在 Toric Code 上，存在两类非平凡环路（逻辑算子）：
- **水平环路**（缠绕环面一个方向）
- **竖直环路**（缠绕环面另一个方向）

但有一条重要性质：**两条水平（或竖直）的逻辑算子会互相抵消为单位算符**——因为两条平行路径可以由一排稳定子叠加出来。因此实际上只有两类非平凡环路需要考虑。

## 6. 旋转表面码（Rotated Surface Code）

旋转表面码将 qubit 编码在格点上（而非边上），得到更高效的编码：

$$
[[L^2, 1, L]]
$$

- $L^2$ 个物理 qubit
- 1 个逻辑 qubit
- 码距 $L$

这比标准表面码的 qubit 数量更少，是实际实验中更常用的形式。

## 7. 解码方法

### 7.1 极大似然解码（Maximum Likelihood Decoding）

- 给每两个 syndrome 异常点之间加上权重（曼哈顿距离）
- 用**最小权重匹配**（Minimum Weight Perfect Matching, MWPM）找到最可能的错误图样
- 如果纠错操作与错误完美匹配，或者正好补成稳定子 → **纠错成功**
- 如果合成出来的是一个逻辑运算符 → **纠错失败**

> [!warning] 解码失败的条件
> 解码器不是万能的。当错误密度太高，使得两个端点之间的最短路径恰好形成一个逻辑算子时，解码器会"修错方向"——把一个逻辑比特翻转了。这就是为什么码距越大（逻辑算子越长），错误越难触发这种灾难性修复。

## 8. 码距与纠错能力

码距 $d$ 决定了纠错能力：
- 可以检测 $d-1$ 个错误
- 可以纠正 $\lfloor (d-1)/2 \rfloor$ 个错误

论文中使用 **distance-5 表面码**：
- 物理 qubit 数：$5^2 + 4^2 = 41$ 个
- 可纠正最多 **2 个物理量子比特的错误**（$\lfloor 4/2 \rfloor = 2$）

$$
p_L \approx \left(\frac{p}{p_{th}}\right)^3 \quad (d=5)
$$

> [!warning] 常见混淆：码距 ≠ 物理 qubit 数
> 码距 $d$ 和物理 qubit 数 $n$ 是不同的概念。$d=5$ 的表面码需要 $d^2 + (d-1)^2 = 41$ 个物理 qubit，但**纠错能力仅由 $d$ 决定**。码距才是衡量保护强度的核心指标。

## 9. 为什么在中性原子平台上特别适合？

[[Optical-Tweezer-Arrays|光镊]] 可以将原子自由排列为二维网格，天然匹配表面码的拓扑结构。通过 [[CZ-Gate]] 实现近邻纠缠操作，配合 [[Rydberg-Blockade]] 实现高保真度两比特门。

> [!info] 二维布局的天然优势
> 光镊阵列可以在二维平面上自由排布原子，无需额外布线层。表面码的拓扑结构恰好是二维的——这意味着**不需要三维集成或长程连接**，用当前技术就能直接实现 surface code 的物理 layout。

## 10. 与颜色码的对比

| 特性 | 表面码 | [[Color-Code\|颜色码]] |
|------|--------|---------|
| 稳定子种类 | 2 种（X + Z） | 3 种（按面颜色分） |
| 顶点度数 | 不固定 | 必须为 3 |
| 横向门 | 需要 magic state | 天然支持 Clifford 门 |
| 连接性要求 | 近邻 | 近邻但需 3-着色 |
| 实验复杂度 | 较低 | 较高 |
| 逻辑比特数（球面） | $k=1$ | $k=0$（需挖洞） |
