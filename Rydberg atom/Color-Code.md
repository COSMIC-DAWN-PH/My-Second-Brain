# 颜色码 (Color Code)

> 来源：高健硕，"Surface Code VS Color Code"，2025
> 本笔记介绍颜色码的基本定义、稳定子结构、与 [[Surface-Code|表面码]] 的区别。颜色码的核心特征是**编码图的每个顶点度数为 3，且面可以 3-着色**，这使得它天然支持横向 Clifford 门。

---

## 1. 什么是颜色码？

颜色码是另一类**拓扑量子纠错码**，与 [[Surface-Code|表面码]] 共享相同的稳定子形式理论基础，但在编码图结构上有本质区别。

> [!tip] 物理直觉
> 表面码用"棋盘格"（方块 + 十字）区分 X 和 Z 稳定子。颜色码用**红、绿、蓝三种颜色**区分三种面——每种颜色对应一种稳定子测量类型。这就像交通灯：红灯停、绿灯行、黄灯注意，三种颜色编码三种不同的"检测模式"。

## 2. 编码图结构

颜色码基于的编码图满足两个关键条件：

1. **每个顶点的度数为 3**（每个点恰好连 3 条边）
2. **所有面可以 3-着色**（相邻面颜色不同）

> [!warning] 与表面码的关键区别
> 表面码的稳定子以**方块 vs 十字**区分类型。颜色码的稳定子以**红/绿/蓝三种颜色**区分。这意味着颜色码的稳定子种类是 3 种而非 2 种，代价是编码图的约束更严格（必须 3-着色）。

## 3. 稳定子结构

颜色码的稳定子定义在面（face）上：

- **红色面**上的 X 和 Z 稳定子
- **绿色面**上的 X 和 Z 稳定子
- **蓝色面**上的 X 和 Z 稳定子

每种颜色的面上可以分别做 X 型或 Z 型稳定子测量。由于相邻面颜色不同，这种颜色区分天然保证了稳定子之间的对易关系。

> [!info] 收缩格点与同伦类
> 可以在红色面中心放一个点作为"收缩格点"，然后将相邻的收缩格点连接起来。这些连线形成的操作算符按同伦类（homotopy class）分类——通过稳定子可以相互转换的两个操作属于同一同伦类。

## 4. 逻辑比特数

通过欧拉示性数（Euler characteristic）$\chi = V - E + F$ 可以推导颜色码的逻辑比特数：

由于顶点度数为 3：$3V = 2E$

$$
k = n - m = V - (2F - 4) = 4 - 2\chi
$$

| 拓扑 | $\chi$ | 逻辑比特数 $k$ |
|------|--------|----------------|
| 球面（无孔） | 2 | $k = 0$（不编码任何逻辑比特） |
| 挖一个孔（圆盘） | 1 | $k = 2$ |
| 环面 | 0 | $k = 4$ |

> [!tip] 为什么球面上 $k=0$？
> 颜色码在球面上的稳定子刚好铺满整个空间，没有"多余"的自由度留给逻辑编码。必须在球面上**挖洞**（引入边界）才能产生逻辑比特。

## 5. Steane 码：挖洞后的经典例子

在球面上**挖去一个格点及周边的三个面**（红、绿、蓝各一个），就得到著名的 **Steane 码**：

$$
[[7, 1, 3]]
$$

- 7 个物理 qubit
- 1 个逻辑 qubit
- 码距 3

> [!info] Steane 码的历史地位
> Steane 码是最早被提出的量子纠错码之一，也是颜色码最简单的实例。它的对称性很高（三维旋转群），并且天然支持横向的 Clifford 门（Hadamard、CNOT、相位门）。

## 6. 与表面码的对比

| 特性 | [[Surface-Code\|表面码]] | 颜色码 |
|------|-----------------|--------|
| 编码图 | 任意二维网格 | 必须 3-着色，顶点度数 = 3 |
| 稳定子种类 | 2 种（X + Z） | 3 种（按面颜色分） |
| 横向 Clifford 门 | ❌ 需要 magic state | ✅ 天然支持 |
| 实验复杂度 | 较低 | 较高（颜色约束） |
| 解码复杂度 | MWPM（高效） | 相对复杂 |
| 球面上 $k$ | 1 | 0（需挖洞） |
| Steane 码 | 不直接对应 | 直接对应 $[[7,1,3]]$ |

> [!question] 颜色码的优势在哪？
> 颜色码的最大优势是**横向 Clifford 门**——在表面码中，Clifford 门需要通过 magic state injection 间接实现，而颜色码可以直接在编码空间中施加。这在容错量子计算中有重要意义，因为它减少了额外的资源开销。

## 7. 与中性原子平台的关系

[[Optical-Tweezer-Arrays|光镊阵列]] 可以自由排列原子位置，理论上可以实现颜色码所需的 3-着色图结构。但相比 [[Surface-Code|表面码]]，颜色码的实验要求更高：
- 需要更复杂的原子排布（3-着色约束）
- 需要更多种类的稳定子测量

在中性原子平台上，表面码目前是更实际的选择（Bluvstein et al. 2026 使用 $[[7,1,3]]$ 和 $[[16,6,4]]$ 等码）。但颜色码的横向门优势使其成为长期目标之一。
